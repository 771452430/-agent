# 支持问题 Agent 当前功能基线

这份清单用于支持问题 Agent LangGraph 改造前后的功能核对。

## 1. 配置与接入

- 支持飞书多维表格地址校验、预览、字段发现、写回验证、待分析筛选预览
- 支持问题列、答案列、链接列、进度列、模块列、登记人列、反馈列、置信度列、命中数列映射
- 支持“模块 -> 支持人 userId”规则与兜底负责人
- 支持从登记人列或域名/邮箱类字段解析登记人账号
- 支持 digest 收件人配置与案例审核开关
- 支持知识范围配置、模型配置、轮巡开关与周期设置

## 2. 主运行链路

- 读取飞书表并筛选 `待分析` / `失败待重试`
- 单行开始处理前回写 `分析中`
- 问题为空时写回失败说明并转 `失败待重试`
- 无命中时写回无命中说明并转 `待人工确认`
- 命中后生成答案、计算命中数、评估置信度
- 低置信度时转 `待人工确认`
- 成功时写回 AI 解决方案、相关链接、置信度、命中数与 `AI分析完成`
- 异常时写回失败摘要并转 `失败待重试`
- 支持相似案例辅助上下文
- `no_change` 时仍会执行反馈同步

## 3. 反馈同步与通知

- 全表 upsert `support_issue_feedback_facts`
- 对关键字段变化追加 `support_issue_feedback_history`
- `待人工确认` 记录支持人补通知
- `人工确认完成` 记录登记人完成通知
- 通知失败不影响主业务状态
- 通知结果写入 `support_issue_notification_events`

## 4. 案例候选与汇总

- 采纳 / 修改后采纳的记录可沉淀为案例候选
- 候选案例支持保存编辑、审核通过并写入知识库
- 支持立即汇总与周期汇总两种 digest
- 汇总包含处理量、AI完成、待人工确认、无命中、失败、采纳/驳回、Top 分类、无命中主题、重点样本、知识缺口建议
- digest 通过 SMTP 邮件发送，并记录 `support_issue_digest_runs`

## 5. 调度与页面展示

- 后端调度器负责到点触发 `run_agent` 与 `run_digest`
- 前端页面可创建/编辑 Agent、立即运行、立即汇总、反馈同步
- 页面展示运行记录、单行处理结果、效果分析、案例候选入口、digest 记录
- 支持案例候选独立页面审核与发布
